# Contributing to VolleyTrack

Thank you for your interest in contributing! We welcome improvements, bug fixes, and new features.

## How to Contribute

### 1. Fork & Clone
1. Fork the repository.
2. Clone it locally:  
   ```bash
   git clone https://github.com/EHB-MCT/dev-v-2025-wdm-Portfolio-starter.git

### 2. Branching
- Always create a feature branch:
    git checkout -b feature/your-feature-name

### 3. Code Standards
- Use functional React components with hooks.
- Name files and components consistently (PascalCase for components, kebab-case for CSS/files).
- Keep code clean and readable; comment when needed.

### 4. Commits
- Write clear, concise commit messages.
- Example:
    feat(LiveMatch): Add player timing chart to LiveMatch analysis

### 5. Testing
- Test your code locally.
- Ensure new functionality does not break existing features.

### 6. Testing
- Push your branch to your fork.
- Open a PR against main.
- Describe your changes clearly.
- Reference any related issues.

### 7. Issues & Feedback
- Report bugs or suggest features via the Issues tab.
- Be descriptive and include steps to reproduce problems.
- Respect maintainersâ€™ time and be patient for feedback.

### Thank You 
Your contributions improve VolleyTrack for everyone. Every PR, bug fix, or suggestion is appreciated!
